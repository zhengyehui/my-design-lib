3

启动开发服务器

本地启动开发服务器，支持热重载：

bash 
# 安装 DevDocs CLI
npm install -g @devdocs/cli

# 初始化新项目
devdocs init my-project --template typescript

# 进入项目目录
cd my-project

# 启动开发服务器
devdocs dev --port 3000
ℹ️
系统要求：Node.js >= 18.0，npm >= 9.0 或 pnpm >= 8.0。推荐使用 Node.js 20 LTS 版本。

配置文件

DevDocs 使用 devdocs.config.ts 作为主配置文件。以下是核心配置项：

typescript 
// devdocs.config.ts
import { defineConfig } from '@devdocs/core'

export default defineConfig({
  project: {
    name: 'My App',
    version: '1.0.0',
  },

  database: {
    adapter: 'postgresql',
    url: process.env.DATABASE_URL,
    pool: { min: 5, max: 20 },
  },

  auth: {
    providers: ['github', 'google'],
    session: { maxAge: 7 * 24 * 60 },
  },

  plugins: [
    analytics(),
    rateLimit({ max: 1000, window: '1m' }),
    cache({ driver: 'redis' }),
  ],
})

API 参数参考

defineConfig() 接受以下核心配置项：

参数 类型 默认值 说明
name 必填 string — 项目名称，用于日志和监控面板显示
database.adapter string sqlite 数据库适配器：sqlite | postgresql | mysql
database.url string — 数据库连接字符串，支持环境变量
auth.providers string[] [] OAuth 认证提供商列表
auth.session.maxAge number 604800 Session 过期时间（秒），默认 7 天
plugins Plugin[] [] 插件数组，按注册顺序执行
cache.driver string memory 缓存驱动：memory | redis | memcached
💡
性能提示：在生产环境中建议使用 PostgreSQL + Redis 组合，可以获得最佳的查询性能和缓存效果。SQLite 仅适用于开发和测试环境。
⚠️
重要：从 v2.x 升级到 v3.x 时，配置文件格式有重大变更。请参考 迁移指南 进行升级。
