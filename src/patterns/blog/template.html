从零搭建一个 AI 原生设计系统

2026-04-19 · Weta · 阅读 8 分钟

最近半年,我们团队把传统的组件库重构成了 AI 原生的设计系统。这篇文章分享一下设计决策和踩过的坑。

为什么传统组件库对 AI 不友好?

传统组件库(React / Vue / Svelte)有几个天然劣势: 框架锁定、API 隐式、依赖链复杂。AI 生成时需要正确 import、调用 hook、处理 state,一步错就全错。

最好的 AI 体验是: AI 不需要思考,复制粘贴就能用。

我们的三个核心决策

第一,抛弃框架,回到纯 HTML + CSS。第二,提供 MCP Server,让 AI 直接调用组件。第三,写 llms.txt 决策树,告诉 AI "什么情况用什么组件"。

// MCP 调用示例 get_component("chat-bubble", "streaming", "snippet") → 返回可粘贴运行的完整 HTML

效果如何?

接入后 3 个月,AI 生成 UI 的一次通过率从 40% 提升到 82%。完整源码已开源,欢迎给个 Star。
